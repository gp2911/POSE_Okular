281  // There is a major problem with dvips, at least 5.86 and lower: the
282  // arguments of the option "-pp" refer to TeX-pages, not to
283  // sequentially numbered pages. For instance "-pp 7" may refer to 3
284  // or more pages: one page "VII" in the table of contents, a page
285  // "7" in the text body, and any number of pages "7" in various
286  // appendices, indices, bibliographies, and so forth. KDVI currently
287  // uses the following disgusting workaround: if the "options"
288  // variable is used, the DVI-file is copied to a temporary file, and
289  // all the page numbers are changed into a sequential ordering
290  // (using UNIX files, and taking manually care of CPU byte
291  // ordering). Finally, dvips is then called with the new file, and
292  // the file is afterwards deleted. Isn't that great?
293 
294  // A similar problem occurs with DVI files that contain page size
295  // information. On these files, dvips pointblank refuses to change
296  // the page orientation or set another page size. Thus, if the
297  // DVI-file does contain page size information, we remove that
298  // information first.

301  // the original file, or a temporary file with a new numbering.
